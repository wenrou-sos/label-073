## 1. 产品概述

租车门店运营管理面板——面向租车门店店长和店员的日常运营管理工具，覆盖今日经营指标监控、车辆预约日历、还车验车、违章处理、门店调车等核心场景，实现门店运营数字化闭环。

- 解决店长日常运营信息不透明、手工记录效率低、违章匹配困难等痛点
- 目标用户：租车门店店长（全局监控与审批）、店员（还车验车与违章处理）

## 2. 核心功能

### 2.1 用户角色

| 角色 | 核心权限 |
|------|----------|
| 店长 | 查看全部门店指标、审批调车申请、发送违章通知、管理违章 |
| 店员 | 还车验车操作、录入里程/油量/损伤、查看订单 |

### 2.2 功能模块

1. **运营仪表盘**：今日关键指标卡片 + 未来7天预约日历视图
2. **还车管理**：还车验车流程，里程录入、油量/电量拍照、损伤标记与费用估算
3. **违章管理**：违章数据导入、订单匹配、通知发送、状态跟踪
4. **调车管理**：门店间调车申请、确认、调车单生成与记录

### 2.3 页面详情

| 页面名称 | 模块名称 | 功能描述 |
|----------|----------|----------|
| 运营仪表盘 | 今日指标卡片 | 展示今日应取车辆数、今日应还车辆数、当前在库可用车辆、待处理违章数，超阈值标红 |
| 运营仪表盘 | 预约日历视图 | 未来7天各车型预约情况日历，绿色（≥3辆）、黄色（1-2辆）、红色（已约满），点击查看订单列表 |

| 还车管理 | 损伤标记 | 车辆俯视图标记新损伤位置（红点），选择损伤类型（划痕/凹陷/掉漆/裂纹），自动估算维修费 |
| 违章管理 | 违章列表 | 展示所有违章记录，包含匹配状态、租客信息 |

| 调车管理 | 调车确认 | B店确认调车申请，生成调车单 |
| 调车管理 | 调车记录 | 查看历史调车单，含调出/调入店、时间、经手人 |

## 3. 核心流程

**还车验车流程**：店员选择待还车订单 → 录入还车里程 → 检查油量/电量并拍照上传 → 对比取车损伤照片 → 在俯视图标记新损伤位置 → 选择损伤类型 → 系统自动估算维修费用 → 确认还车，额外费用加入订单

**违章处理流程**：导入交管违章数据 → 系统按车牌+时间段自动匹配订单和租客 → 店长审核匹配结果 → 一键发送通知给租客 → 租客自行处理或选择门店代办 → 标记已处理

**调车流程**：A店发现缺车 → 发出调车申请（车型+数量） → B店在面板看到申请 → B店确认 → 系统生成调车单 → 记录调出店、调入店、调出时间、经手人

## 4. 用户界面设计

### 4.1 设计风格

- **主色调**：深蓝灰 (#1e293b) 为底色，搭配亮蓝 (#3b82f6) 作为操作强调色，红 (#ef4444) 用于警告/超阈值，绿 (#22c55e) 用于正常/充足
- **按钮风格**：圆角 6px，扁平化按钮，主要操作用实心蓝色，危险操作用红色
- **字体**：正文使用系统字体栈，数字指标使用等宽字体突出数据感
- **布局风格**：左侧固定导航栏 + 右侧内容区，表格和日历为主视图
- **图标**：使用 lucide-vue-next 图标库，线性风格

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 运营仪表盘 | 指标卡片 | 4个等宽卡片横排，数字大号加粗，超阈值时卡片边框变红+数字变红 |
| 运营仪表盘 | 日历视图 | 7列×N行表格（列为日期、行为车型），单元格背景色按库存状态着色，点击弹出订单列表抽屉 |
| 还车管理 | 里程油量 | 表单输入框+文件上传组件，双列布局 |
| 还车管理 | 损伤标记 | SVG车辆俯视图居中，可点击放置红点，右侧损伤类型选择和费用列表 |
| 违章管理 | 违章列表 | 数据表格，含状态标签、操作按钮，顶部筛选区 |
| 违章管理 | 导入 | 模态框，粘贴或上传违章数据 |
| 调车管理 | 调车列表 | 卡片式列表+状态流转，新建申请用模态框表单 |

### 4.3 响应式

- 桌面优先，最小宽度 1280px
- 侧边栏可折叠，内容区自适应
- 日历视图在小屏下可横向滚动
